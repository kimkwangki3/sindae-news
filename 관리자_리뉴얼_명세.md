# 신대신문 — 관리자(백오피스) 리뉴얼 명세 (VSCode 작업용)
_목표: 슈퍼관리자가 사이트의 모든 콘텐츠·사람·광고·통계를 한 곳에서 열람하고 관리._
_기존 빌드 위에 적용. `개발계획서.md` · `기자기능_추가명세.md` · `db/회원등급_설계.md` 함께 참고._

## 0. 문제 (현재 상태)
관리자 화면에 기사·댓글·신고·광고·회원만 있고 **기자 승인/관리, 지역단체, 게시판, 나눔마켓, 상권/업체, 제보, 광고신청, 통계**가 빠져 있음.
→ 슈퍼관리자가 전체를 통제할 수 없음. **전 영역 백오피스로 리뉴얼.**

## 1. 원칙
- **슈퍼관리자 = 전권**: 모든 게시물(기사·댓글·게시판·나눔·상권·단체)을 열람·수정·숨김·삭제 가능.
- **승인이 필요한 모든 것**을 큐로 모아 처리(기자/업체/단체/광고/준기자 기사).
- **현황 가시성**: 방문자·인기 기사·인기 게시글을 대시보드와 통계에서 즉시 확인.
- **안전장치**: 모든 관리 행위는 **감사 로그(audit log)** 에 기록. 권한은 등급으로 제한.

## 2. 관리자 등급
| 등급 | 범위 |
|---|---|
| **superadmin** | 전체 관리 + 관리자 임명/회수 + 사이트 설정 + 감사로그 |
| **admin(운영자)** | 콘텐츠·신고·승인·회원 관리(설정/권한/로그 제외) |
> `profiles.role` 사용. superadmin만 admin 임명·해제 가능.

## 3. 관리자 네비게이션 IA (사이드바, 그룹화)
```

  - 기자 신청·관리      /admin/reporters              [배지: 신청수]
  - 정정보도 관리       /admin/corrections
```

## 4. 페이지별 명세
### 대시보드 `/admin`
- 카드: 오늘 방문자·PV / 신규 회원 / 발행 기사 / **처리 대기 합계**(기자신청+업체+단체+광고+준기자기사+신고+제보)
- "처리할 일" 위젯: 큐별 대기 건수 + 바로가기
- 인기 콘텐츠 TOP: 오늘 조회 1~5위 기사, 인기 게시글
- 최근 활동: 최근 발행/신고/가입

### 통계·분석 `/admin/analytics`
- 방문: 일/주/월 방문자·PV 그래프, **유입 경로**(카카오/검색/직접), 시간대 분포
- 콘텐츠: **기사별**(조회·순방문·읽음률·체류·좋아요·댓글), **게시글별** 인기, 카테고리별 합계
- 회원: 가입 추이, 활성 사용자, 등급 분포
- 기간/카테고리 필터 + CSV 내보내기

### 기사 관리 `/admin/articles`
- 목록(제목·카테고리·작성기자·상태·조회·좋아요·댓글·발행일) + 검색/필터(상태·카테고리·기자)
- 액션: 수정·발행/비공개·삭제·대표기사 지정
- 행 클릭 → 기사별 상세 통계

### 기사 승인 큐 `/admin/articles/review`
- **준기자 pending 기사**: 미리보기·서약여부 확인 → 「게시 승인」/「반려(사유)」

### 기자 신청·관리 `/admin/reporters`
- 신청 대기: 서약 확인 → 승인(등급 junior 기본)/반려
- 활동 기자: 등급(정기자/준기자) 지정·승격·강등, 작성기사 수, 권한 정지
- ※ 상세는 `기자기능_추가명세.md`

### 정정보도 관리 `/admin/corrections`
- 오보·정정 요청 접수 → 기사에 정정 안내 게시(언론 법적 의무)

### 댓글 / 게시판 / 나눔마켓 관리
- 공통 패턴: 전체/신고/숨김 필터, 검색, 행 액션(노출·숨김·삭제·고정), 작성자 바로가기
- 게시판: 공지 고정, 분류 관리 / 나눔마켓: 상단고정·완료처리

### 제보함 `/admin/tips`
- 신규/확인/기사화/처리완료. 「기사화」→ 기사 작성 프리필.

### 신고 관리(통합) `/admin/reports`
- 유형(기사·댓글·업체·단체·나눔·게시판) 필터, 신고수 누적 정렬, 숨김/삭제/경고/무시, 신고자·시간 로그.

### 상권/업체 · 지역단체 관리
- 등록 승인/반려, 운영진/회원 권한, 게시물(홍보글·단체소식) 관리, 상단 노출.
- 업체는 **1인 1개** 규칙 확인.

### 광고 배너 / 광고 신청
- 배너: 슬롯별 이미지·링크 교체, 슬롯 추가, 자동광고(애드핏/애드센스) 설정.
- 신청: 접수→승인·게재/반려/중단.

### 회원 관리 `/admin/members`
- 검색(닉네임·연락처), 등급/소속(업체·단체) 표시, 닉네임 강제변경, **정지(사유·기간)·해제·탈퇴**.
- 회원 상세: 작성 기사/댓글/게시글·신고 이력.

### 설정 (superadmin)
- 관리자·권한: admin 임명/회수, 슈퍼관리자 위임.
- 카테고리·광고슬롯 관리, 법적 페이지(발행인·청소년보호·윤리강령·정정보도·개인정보) 편집.
- 감사 로그: 누가 무엇을 언제 변경했는지 조회.

## 5. DB 추가 (마이그레이션)
```sql
-- 관리 행위 감사 로그
create table if not exists admin_audit_logs (
  id          bigint generated always as identity primary key,
  actor_id    uuid references profiles(id),       -- 수행 관리자
  action      text not null,                      -- approve_reporter / hide_comment / delete_article ...
  target_type text,                               -- article/comment/business/...
  target_id   uuid,
  memo        text,
  created_at  timestamptz not null default now()
);
create index on admin_audit_logs (created_at);
create index on admin_audit_logs (actor_id);

-- 정정보도
create table if not exists corrections (
  id          uuid primary key default gen_random_uuid(),
  article_id  uuid references articles(id),
  reason      text,
  body        text,                               -- 정정 내용
  status      review_status not null default 'pending',
  created_at  timestamptz not null default now()
);
```
> 기자 2등급·기사 pending·서약 컬럼은 `기자기능_추가명세.md`의 마이그레이션 사용.

## 7. 공통 UX 규칙
- 모든 목록: 검색 + 필터 + 페이지네이션 + 정렬, 위험 액션(삭제·정지)은 확인 모달.
- 일괄 처리(체크박스 다중 선택 → 숨김/삭제).
- 모든 승인/숨김/삭제/정지 → `admin_audit_logs` 기록.
- 사이드바 대기건수 배지는 서버에서 카운트.
- 권한 가드: 라우트 미들웨어(`/admin/*` 는 admin 이상, `/admin/{staff,settings,logs,legal}` 는 superadmin) + RLS.
