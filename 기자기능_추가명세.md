# 신대신문 — 기자 기능 추가 명세 (VSCode 작업용)
_기존 빌드 위에 추가하는 기능. `개발계획서.md` · `db/회원등급_설계.md`와 함께 적용_

## 요약 (이번에 추가할 것)
1. **기자 전용 작성 공간** — 기자가 기사를 쓰는 페이지
2. **내 기사 관리/통계** — 내가 쓴 기사의 조회수·좋아요·댓글 확인
3. **기자 2등급제**
   - **정기자(senior)** — 승인 없이 작성 즉시 발행
   - **준기자(junior)** — 관리자 승인 후 발행
4. **기사별 책임 서약** — "본 기사의 책임은 회사가 아닌 작성 기자 본인에게 있다"를 발행 시 반드시 서약(체크+기록)

## 1. 데이터 모델 변경 (마이그레이션 SQL)
> 이미 DB가 생성돼 있으므로 ALTER 로 추가. Supabase SQL Editor 에서 실행.

```sql
-- 기자 등급
create type reporter_level as enum ('junior','senior');   -- 준기자 / 정기자
alter table profiles add column if not exists reporter_level reporter_level;
--   role='reporter' 인 사람만 의미. 승인 시 관리자가 junior/senior 지정

-- 기사 상태에 '승인대기' 추가 (준기자 발행 흐름)
alter type article_status add value if not exists 'pending';
--   draft → (준기자) pending → published / (정기자) 곧장 published

-- 기사별 책임 서약 + 검수 기록
alter table articles add column if not exists pledge_ack    boolean not null default false;
alter table articles add column if not exists pledge_ack_at timestamptz;
alter table articles add column if not exists reviewed_by   uuid references profiles(id);
alter table articles add column if not exists reviewed_at   timestamptz;
```
※ `alter type ... add value` 는 트랜잭션 안에서 다른 구문과 같이 실행 못 함 → 단독 실행.

## 2. 발행 흐름 (핵심 규칙)
| 작성자 | 저장(임시) | 제출 시 상태 | 노출 |
|---|---|---|---|
| 정기자(senior) | draft | **published** (즉시) | 바로 사이트 노출 |
| 준기자(junior) | draft | **pending** (승인요청) | 관리자 승인 후 published |
| 관리자 | draft | published | 즉시 |

- 발행/승인요청 버튼 누를 때 **서약 체크 필수**: `pledge_ack=true`, `pledge_ack_at=now()` 저장. 미체크 시 발행 불가.
- 준기자 기사 승인 시: 관리자 `status='published'`, `published_at=now()`, `reviewed_by`, `reviewed_at` 기록.
- 정지(`is_suspended`)된 기자는 작성·발행 모두 불가(읽기만).

## 3. 추가 페이지 (라우트)
### 기자 전용 `/reporter` (role=reporter 또는 admin 접근)
- `/reporter` — **기자 대시보드**
  - 요약 카드: 내 기사 수 · 총 조회수 · 총 좋아요 · 총 댓글 · (준기자면) 승인대기 수
  - 내 등급 표시(정기자/준기자) + 등급별 발행 방식 안내
  - 최근 내 기사 리스트(상태·조회·좋아요·댓글)
- `/reporter/write` — **기사 작성**
  - 시안 C(기사 작성 에디터) 재사용: 제목·본문·분류·슬러그·대표이미지
  - 하단 **서약 박스**(필수 체크): "본 기사의 모든 법적·윤리적 책임은 회사(DSBH)가 아닌 작성 기자 본인에게 있습니다."
  - 버튼: 정기자→「발행」, 준기자→「승인 요청」, 공통「임시저장」
- `/reporter/articles` — **내 기사 목록**
  - 상태 필터(전체/발행/승인대기/임시), 각 행에 조회·좋아요·댓글, 수정 링크
- `/reporter/articles/[id]/stats` — **기사별 상세 통계**
  - 조회수·순방문(IP)·평균 읽음률(scroll_pct)·평균 체류(dwell_ms)·좋아요/싫어요·댓글 목록
  - 기간별(일/주/월) 추이

### 관리자
- `/admin/articles` 에 **「승인 대기」 필터** 추가(= 준기자 pending 기사 큐)
  - 행: 제목·작성기자·등급·서약여부·미리보기 → 「게시 승인」/「반려(사유)」
- 기자 관리(L)에서 **등급 지정/변경**: 승인 시 junior 기본 → 활동 보고 senior 승격, 강등도 가능

## 4. 권한 로직 (`lib/permissions.ts` 보강)
```ts
// 기사 작성 가능?
function canWriteArticle(u) {
  return !u.is_suspended && (u.role === 'reporter' || isAdmin(u));
}

// 제출 시 결정되는 상태
function articleStatusOnSubmit(u) {
  if (isAdmin(u) || u.reporter_level === 'senior') return 'published'; // 즉시
  return 'pending';                                                    // 준기자: 승인대기
}

// 발행/제출 가드: 서약 미동의면 차단
function canSubmitArticle(u, form) {
  return canWriteArticle(u) && form.pledge_ack === true;
}

// 준기자 기사 승인은 관리자만
function canApproveArticle(u) { return isAdmin(u); }
```
- RLS도 이중 적용: 기사 insert/update 는 `author_id = auth.uid()`(기자 본인) 또는 관리자. `status='published'` 로의 전환은 senior 본인 or 관리자만 허용하도록 정책 분리(준기자는 published 직접 설정 불가, pending 까지만).

## 6. 통계 쿼리 예시 (참고)
```sql
-- 기자별 합계
select count(*) filter (where status='published') as 발행수,
       coalesce(sum(view_count),0) as 총조회
from articles where author_id = :uid;

-- 기사별 평균 읽음률/체류
select avg(scroll_pct) 읽음률, avg(dwell_ms) 체류ms
from article_views where article_id = :aid;
```
